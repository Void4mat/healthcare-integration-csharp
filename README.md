# Healthcare Integration — C# & FHIR R4

A hands-on portfolio of healthcare system integration exercises 
built using C# and the Firely .NET SDK, connecting to the 
HAPI FHIR R4 public test server.

Built to develop practical skills in healthcare integration 
patterns relevant to hospital and clinical system environments.

---

## Tech Stack

- C# / .NET 8
- Firely .NET SDK (Hl7.Fhir.R4)
- HAPI FHIR R4 Public Test Server
- nHapi (HL7 v2 parsing — Week 4)
- Visual Studio Community 2022

---

## Projects

### Week 1 — FHIR GET
Connect to a FHIR R4 server and read Patient resources.
Covers resource structure, identifiers, and search parameters.

### Week 2 — FHIR POST (coming)
Create Patient resources on a FHIR R4 server.
Covers resource creation, MRN identifiers, and server responses.

### Week 3 — Admission Flow (coming)
Simulate a full patient admission integration.
Covers Patient to Encounter linking and relationship queries.

### Week 4 — HL7 v2 Parsing (coming)
Parse raw HL7 ADT^A01 messages and map to FHIR R4 resources.
Covers the bridge between legacy HL7 v2 and modern FHIR standards.

---

## Why This Matters

Healthcare integration requires understanding both legacy HL7 v2 
messaging and modern FHIR R4 REST APIs. This project bridges both, 
mirroring real-world integration patterns used in hospital 
information systems and clinical platforms.
